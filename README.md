# J.A.R.V.I.S. Mark I 🤖

A personal AI assistant that analyzes messy, unstructured user prompts and organizes them into structured To-Do tasks and calendar events. The project implements an advanced **Dynamic Model Routing** pattern to optimize API limits and costs, alongside **Time Anchoring** (system-level time injection) to accurately resolve relative dates (e.g., "tomorrow", "next week", "in 2 hours").

## 🧠 Core Features

### 1. Gatekeeper and Dynamic Model Routing (Model Router)
Protects API limits and optimizes expenses using a two-stage evaluation pipeline:
*   **Gatekeeper:** First evaluates the user prompt using a fast and cost-effective model (`gemini-1.5-flash` or `gpt-4o-mini`). It returns a structured JSON with a boolean flag `is_complex` and a brief description of the `reasoning`.
*   **Executor:**
    *   If `is_complex: False` (e.g., adding a simple single chore) -> the final JSON extraction is generated by the same cheap model.
    *   If `is_complex: True` (e.g., scheduling conflicts, multi-step planning, or complex dates) -> the agent **dynamically swaps** the model to a more powerful, premium tier (`gemini-1.5-pro` or `gpt-4o`) to perform the advanced reasoning.

### 2. Time Anchoring (System Time Awareness)
Before each LLM call, the current system time is injected into the System Prompt/Instruction in the format `Aktualny czas systemu: YYYY-MM-DD HH:MM`. This allows the AI models to interpret relative date and time specifications such as *"tomorrow"*, *"next Monday"*, or *"in 2 hours"* relative to the host system clock.

### 3. Automated Mock Integrations
Once the agent finishes structuring the data, the FastAPI backend automatically invokes mock integrations:
*   **Google Calendar** (`GoogleCalendarMock.create_event()`) - prints a beautifully formatted event box directly in the server console.
*   **Google Tasks** (`GoogleTasksMock.create_task()`) - prints a formatted task details box in the server console.

## 🛠️ User Setup & Deployment Guide

Follow these steps to fully configure, install, and run the personal assistant on your local machine:

### Step 1: Configure Environment API Keys in `.env`
1.  Copy the environment template file:
    ```bash
    cp .env.example .env
    ```
2.  Open `.env` in your editor and choose your active provider with `AI_PROVIDER`:
    *   **Option A: Google Gemini (Recommended)**
        *   Ensure `AI_PROVIDER=gemini` is set.
        *   Obtain a free or paid API Key from [Google AI Studio](https://aistudio.google.com/).
        *   Paste the key in: `GEMINI_API_KEY=your_actual_key_here`.
        *   By default, the assistant uses `gemini-1.5-flash` for the cheap gatekeeper and `gemini-1.5-pro` for the premium executor.
    *   **Option B: OpenAI**
        *   Set `AI_PROVIDER=openai`.
        *   Obtain an API key from the [OpenAI Platform](https://platform.openai.com/).
        *   Paste the key in: `OPENAI_API_KEY=your_actual_key_here`.
        *   By default, the assistant uses `gpt-4o-mini` for the gatekeeper and `gpt-4o` for the premium executor.

### Step 2: Prepare Python Virtual Environment & Dependencies
Run the following commands in your terminal inside the project's root folder:
```bash
# 1. Create a Python virtual environment
python3 -m venv venv

# 2. Activate the virtual environment
source venv/bin/activate

# 3. Upgrade package installer and install requirements
pip install --upgrade pip
pip install -r requirements.txt
```

### Step 3: Launch the FastAPI Backend Server
With the virtual environment active, start the FastAPI API server using uvicorn:
```bash
uvicorn server:app --reload
```
*The server will boot and run on `http://127.0.0.1:8000`. If API keys are missing in `.env`, a critical startup warning will be logged to the console, but the server will continue running. Error codes will only be thrown when a request is actively processed.*

### Step 4: Launch the Streamlit Frontend App
Open a separate terminal window, navigate to the project directory, activate the environment (`source venv/bin/activate`), and start the frontend:
```bash
streamlit run app.py
```
*The frontend web app will automatically launch in your browser at `http://localhost:8501`.*

### Step-by-Step execution flow:
1.  **System Time Context:** The system anchors the local datetime (e.g., `Aktualny czas systemu: 2026-06-23 18:32`).
2.  **Gatekeeper Check:** The gatekeeper (`gemini-1.5-flash` / `gpt-4o-mini`) processes the prompt. Since it contains a timeline ("tomorrow", "later"), legal references ("2018 regulations"), and multiple chores, it outputs `is_complex: True`.
3.  **Model Routing Switch:** The backend detects the complex flag and redirects the request to the high-tier Executor (`gemini-1.5-pro` / `gpt-4o`).
4.  **Parsing & Output Generation:** The executor parses the prompt and generates structured JSON output mapping:
    *   **Calendar Event 1:** Submission of promotion application (assigned to tomorrow, `2026-06-24`).
    *   **Calendar Event 2:** Signing document sheets at school (assigned to tomorrow, `2026-06-24`, scheduled sequentially after Event 1).
    *   **To-Do Task 1:** Curtain washing.
    *   **To-Do Task 2:** Cleaning up the apartment.
